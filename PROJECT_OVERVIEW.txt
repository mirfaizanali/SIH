================================================================================
       CAMPUS INTERNSHIP & PLACEMENT PORTAL — COMPLETE PROJECT OVERVIEW
================================================================================

Tech Stack:
  Backend  : Spring Boot 4.0.5, Java 21, Spring Security 6, JWT (jjwt 0.12.6)
  Frontend : Angular 20.3.0, Standalone Components, Signals, RxJS 7.8
  Database : MySQL 8.x (Hibernate ddl-auto manages schema)
  Cache    : Caffeine (Spring Cache)
  Real-time: STOMP over SockJS (WebSocket)
  Auth     : JWT access token (15 min) + refresh token cookie (7 day, SHA-256 hashed)

================================================================================
                         ROLES & PERMISSIONS
================================================================================

  ADMIN > PLACEMENT_OFFICER > FACULTY_MENTOR > STUDENT
  EMPLOYER (flat peer, no hierarchy with academic roles)

  STUDENT            : Apply to jobs, upload resume, view recommendations, submit reports
  EMPLOYER           : Post jobs/internships, manage applicants, schedule interviews
  FACULTY_MENTOR     : View mentees, review internship reports
  PLACEMENT_OFFICER  : Manage placement drives, verify employers, view analytics
  ADMIN              : Manage users, system config, audit logs (inherits all permissions)

================================================================================
                         KEY BUSINESS FLOWS
================================================================================

1. JOB APPLICATION FLOW
   Student browses jobs → views recommendations → uploads resume →
   applies (CGPA validated) → employer reviews → shortlists →
   schedules interview → records feedback → offers/rejects →
   notifications sent at each step (DB + WebSocket + email)

2. PLACEMENT DRIVE FLOW
   Officer creates drive (links jobs) → broadcasts to students →
   students browse → drives go IN_PROGRESS → interviews during drive →
   officer marks COMPLETED

3. INTERNSHIP FLOW
   Employer posts internship → student applies → enrolled →
   faculty mentor assigned → student submits reports →
   faculty reviews/approves reports

4. RECOMMENDATION ENGINE
   Student requests → loads profile + skills → loads active jobs (cached) →
   ScoringEngine scores all (CGPA weight, skill match, experience, preferences) →
   hard disqualifier: if student.cgpa < job.minCgpa → score=0 →
   returns top N with matchScore

================================================================================
                         SECURITY ARCHITECTURE
================================================================================

  - Stateless JWT (no HTTP sessions)
  - Access token: 15 min, in-memory on frontend (never localStorage)
  - Refresh token: 7-day, httpOnly SameSite=Strict cookie, SHA-256 hashed in DB
  - Token rotation on every refresh (replay attack detection)
  - BCrypt strength=12 for passwords
  - Rate limiting: 100 req/min per user, 20 req/min on /api/auth/login
  - CORS: only localhost:4200 in dev
  - File uploads: MIME whitelist (PDF, DOC, DOCX), max 10MB
  - Audit logging: all mutating requests (POST/PUT/PATCH/DELETE) logged async
  - Never log passwords, tokens, or PII

================================================================================
                         CACHING
================================================================================

  active_jobs          : 15 min TTL, evicted on job status change
  analytics_dashboard  : 15 min TTL, evictable by admin via POST /api/analytics/refresh

================================================================================
                         REAL-TIME NOTIFICATIONS
================================================================================

  Protocol  : STOMP over SockJS at /ws
  Auth      : Bearer token in STOMP CONNECT headers
  Queue     : /user/queue/notifications (per-user)
  Flow      : Save to DB → push via WebSocket → fire-and-forget email

  Triggers  :
    - Application submitted/status changed
    - Interview scheduled
    - New job posted (broadcast)
    - Report feedback
    - Drive announced (broadcast)
